MyOS - 32-bit x86 Operating System Project

Overview:
This is a basic operating system kernel developed for x86 architecture. The project focuses on low-level systems programming, including bootloading, memory-mapped I/O, and hardware driver development.

Current Features:
1. Bootloader:
- Developed in 16-bit Assembly (NASM).
- Handles the transition from Real Mode to 32-bit Protected Mode.
- Loads the kernel into memory.

2. VGA Video Driver (drivers/const.c):
- Direct memory access to 0xB8000.
- Custom printf implementation supporting strings (%s), integers (%d), characters (%c), and pointers (%p).
- Advanced Screen Management: Includes a RAM-based Terminal Buffer that stores a larger history of printed text than the physical screen.
- Automatic Scrolling: Moves the display window within the buffer when the cursor exceeds the screen boundaries.
- Color Support: Full control over foreground and background text colors using bitwise operations.

Project Structure:
- boot/: Assembly source code for the bootloader.
- kernel/: The core C code for the OS kernel.
- drivers/: Hardware-specific drivers (Screen, Constants).
- bin/: Compiled binary files and final OS image (Generated by Makefile).

Compilation and Execution:
The project uses a Makefile for automated building.

Required Tools:
- GCC (with m32 support)
- NASM
- QEMU (for emulation)
- Make

Build Instructions:
1. Clean previous builds: make clean
2. Compile and run: make run

Technical Implementation Details:
- The screen buffer (terminal_buffer) consists of unsigned short values (16-bit).
- Each entry combines an 8-bit ASCII character with an 8-bit attribute byte (color).
- Scrolling is managed by adjusting the current_view_row pointer, allowing for future scroll-back capabilities.
